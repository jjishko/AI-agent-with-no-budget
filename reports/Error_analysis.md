# Rule-based модель для NER

Показательные примеры ошибок:

	   ПРЕДСКАЗАНО: тип='quantity', значение='8705', позиция=49-53
	   ИСТИНА:      тип='model', значение='8705', позиция=49-53
	   КОНТЕКСТ:    Покажи инструкцию по калибровке модели Rosemount **8705** для расходомера

	   ПРЕДСКАЗАНО: тип='symptom', значение='шумит', позиция=19-24
	   ИСТИНА:      тип='symptom', значение='сильно шумит', позиция=12-24
	   КОНТЕКСТ:    Насос Н-904 сильно **шумит** и вибрирует

	  ПРЕДСКАЗАНО: тип='tag', значение='DVC-6200', позиция=42-49
	   ИСТИНА:      тип='model', значение='DVC-6200', позиция=42-49
	   КОНТЕКСТ:    Сравни модели Siemens Sipart PS2 и Fisher **DVC6200** на клапанах

	   ТИП:         action
	   ЗНАЧЕНИЕ:    'Суммируй'
	   ПОЗИЦИЯ:     0 - 8
	   КОНТЕКСТ:    **Суммируй** расходы по четырем измерительным каналам
	   ПРИЧИНА:     Модель не распознала эту сущность

	   ТИП:         action
	   ЗНАЧЕНИЕ:    'Напиши'
	   ПОЗИЦИЯ:     0 - 6
	   КОНТЕКСТ:    **Напиши** инструкцию по переходу на основной насос
	   ПРИЧИНА:     Модель не распознала эту сущность

Наиболее часто модель ошибается при разметке сущностей action, quantity, equipment. Наиболее сложно модели с action, так как действие для ИИ-агента может последовать любым, что затрудняет определение всех вариантов. Более того, не всегда известно, что считать оборудованием, моделью или числом - определяется исключительно по контексту. 

Если учесть эти замечания вместе с полученными метриками (см. NER_metrics.md), вполне уместно использование ML/LLM - подходов для анализа контекста.
